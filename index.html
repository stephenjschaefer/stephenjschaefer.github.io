• Steve's Bits & Bytes

    • January 14th, 2016

    I've finished the initial release of a Ruby on Rails implementation of the 15 position triangle peg solitaire game often found at Cracker Barrel. To implement different data structures valid moves are stored in an array and the peg jumped by those valid moves is stored in a hash, allowing for a list of valid moves for a given board state to be presented to the user for reference while playing. The application also makes use of the RubyTree gem to recursively build a solution tree in order to determine if a given board configuration can be solved such that there is only one peg remaining, providing the user feedback to guide them to a solution while playing. The source is available in my public GitHub repository and I've pushed the application to Heroku and it's available to play at the following URL.

    January 4th, 2016

    I wanted to try out the Heroku platform's MySQL DBaaS add on offered through ClearDB, so I decided to stand up an instance of the popular PHP based Coppermine Photo Gallery. The photo gallery can be accessed via this link however due to Heroku's non-persistent file system that is recreated each time a dyno is restarted no user photos are uploaded yet. I've added the source to GitHub as I plan to modify the code to either store images in the database directly rather than storing a file path reference to the image or instead leverage Amazon S3 for persistent storage of uploaded images. I'm also going investigate what changes would be required in order to make CPG compatible with the latest PHP 7.0.1 release which updates the supported PHP MySQL Drivers, removing the original MySQL extension that CPG currently uses in favor of the new MySQLi extension.

    December 21st, 2015

    I've uploaded the latest source to one of my favorite projects - a Palm OS based OBD-I automotive diagnostic and datalogging application known as "Mitsubishi Motor Corporation datalogger", or MMCd for short. This project combined some of my favorite interests and was my first experience doing mobile development. MMCd was originally authored by Dmitry Yurtaev with additional contributions made by Shawn Poulson, Clinton Battersby and I. MMCd is written in C and is built using Cygwin, prc-tools, and pilrc in a Windows environment. Additional details about the communication protocol used and hardware compatibility are available on the sourceforge project page created by Dmitry. My involvement in the project started with v1.6p, where Clint and I worked together to fix bugs and implement new features which were well received by the 3SI user community as detailed in this thread on 3SI.org.
